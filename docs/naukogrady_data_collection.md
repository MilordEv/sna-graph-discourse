# Сбор корпуса «Наукограды России»

Документ описывает, как был собран тематический корпус для проекта дискурс-графов.  
Скрипт сбора: [`load_data.ipynb`](../load_data.ipynb) (ячейки конфигурации, сбора и построения графа).

Связанные материалы: [`discourse.md`](discourse.md), [`project_summary.md`](project_summary.md).

---

## Цель корпуса

Собрать **узкий доменный** набор русскоязычных текстов про наукограды России: города, НИИ, федеральную поддержку, наукоёмкую продукцию, городскую среду для исследователей — с опорой на материалы из [РБК Тренды](https://trends.rbc.ru/trends/education/60ca120a9a794708a559c080) и свежие новости (в т.ч. присвоение статуса наукограда Долгопрудному, февраль 2026).

Корпус предназначен для:
- отладки конструктора дискурс-графа (вершины, рёбра, PMI);
- промежуточного показа результатов;
- сравнения с GraphRAG / long-context baseline.

## Пайплайн сбора

```
Источники URL  →  Скачивание (newspaper3k)  →  Фильтр длины  →  Фильтр релевантности
       →  NER (Natasha) + YAKE  →  Метаданные  →  Дедупликация по URL  →  documents.json
       →  Граф (co-occurrence по предложениям + PMI + пороги)  →  graphs/naukogrady/
```

## Источники URL

Сбор шёл **только по корпусу `naukogrady`** (`COLLECT["naukogrady"] = True`). Росатом и международное НТС в этом прогоне не собирались.

### 1. Сиды (seed URLs)

Фиксированный список статей РБК Тренды, включая лонгрид-эталон:

- `https://trends.rbc.ru/trends/education/60ca120a9a794708a559c080` — «Как работают наукограды в России»
- ещё 3 статьи из раздела education

**Сиды скачиваются без повторной фильтрации релевантности** — они считаются тематически валидными по определению.

### 2. RBC Trends (скрапинг разделов)

RSS `https://trends.rbc.ru/feeds/rss` отдаёт **404**, поэтому ссылки берутся парсингом HTML:

- `https://trends.rbc.ru/trends/education`
- `https://trends.rbc.ru/trends/tag/science`

Из HTML извлекаются URL вида `/trends/education/{id}` и `/trends/science/{id}`.

### 3. Google News RSS

Поисковые запросы (до 25 результатов на запрос):

- `наукоград Россия`, `наукограды России`
- `Долгопрудный наукоград 2026`, `статус наукограда Долгопрудный`

- `Сколково наукоград исследователи`, `Иннополис наукоград`
- `научно-технологический потенциал наукоград`
- `site:trends.rbc.ru наукоград`, `site:minobrnauki.gov.ru наукоград`
- `Фальков наукоград`

Дополнительно — **по одному запросу на каждый из 12 городов**:

Ссылки вида `news.google.com/rss/articles/...` **раскодируются** в прямые URL источника через `googlenewsdecoder`.

Ленты:

## Фильтрация и отбор

### Минимальная длина

Текст статьи **≥ 400 символов** после очистки. Более короткие отбрасываются (`skipped_short`).

### Релевантность домену (`classify_naukograd`)

Документ принимается, если выполняется **хотя бы одно** условие:

1. **Заголовок:** слова `наукоград`, `наукограды` или `город науки`.
2. **Город + контекст:** название из whitelist городов **и** в первых ~1000 символах есть научные маркеры (`нии`, `технопарк`, `исследовател`, `наукоёмк`, `ран` и т.д.).
3. **Доменный маркер** в начале текста: `наукоград`, `наукоёмк`, `нтп` и др.

**Whitelist городов:** Долгопрудный, Дубна, Обнинск, Королёв, Фрязино, Пущино, Жуковский, Реутов, Троицк, Черноголовка, Сколково, Иннополис, Протвино и др.

### Статистика отсева (один прогон)

При обработке кандидатов (не считая сидов):

| Категория | Количество |
|-----------|------------|
| Сохранено | 218 |
| Слишком короткий текст | 74 |
| Не прошёл фильтр релевантности | 65 |

### Дедупликация

По полю `url`: при дубликате остаётся версия с **большей длиной текста**.

## Обогащение документов

Для каждого принятого документа:

| Поле | Метод |
|------|--------|
| `entities` | Natasha: `Segmenter` + `NewsNERTagger` (PER, ORG, LOC) |
| `keywords` | YAKE, `lan=ru`, n-граммы до 3 слов, top=20 |
| `domain` | всегда `"naukogrady"` |
| `voice_type` | эвристика по URL/заголовку: `media`, `official`, `interview`, `analytics` |
| `text_type` | `news`, `interview`, `speech`, `analytics` |
| `actor` | regex по именам (Фальков, Мишустин, Кириенко, глава РАН и др.) |
| `naukograd_city` | первый город из whitelist, найденный в тексте |
| `relevance_reason` | код правила фильтрации |

Прочие поля: `doc_id` (UUID), `source`, `url`, `date`, `title`, `text`, `authors`, `top_image`.

## Построение графа

Граф строится **отдельно** по корпусу наукоградов (`data/graphs/naukogrady/`), не по старому общенаучному графу.

### Co-occurrence

- Уровень: **предложение** (не весь документ).
- Узлы в предложении: NER-сущности документа, встречающиеся в этом предложении + до 10 ключевиков YAKE, если они есть в предложении.
- Рёбра: неориентированные пары внутри предложения, вес = число совместных предложений.

### Фильтры (чтобы избежать плотного графа n²)

| Параметр | Значение |
|----------|----------|
| Top узлов по степени | 400 |
| Минимальный вес ребра | 2 |
| Минимальный PMI | 5.0 |
| Изолированные узлы | удаляются |

PMI считается по весам рёбер после отбора top-узлов.

**Результат:** 253 узла, 317 рёбер (медиана PMI ≈ 5.75) — порядок величины **2n–3n**, как в плане проекта.

## Ограничения и известные артефакты

1. **Ложные срабатывания по топонимам** — слова «Королёв», «Троицк» и т.п. могут встречаться не в контексте наукограда; частично сглаживается правилом «город + научный маркер».
2. **Преобладание новостей** (`voice_type=media`) — мало интервью и официальных речей; для dropout «экспертов» корпус пока слабый.
3. **Google News** — при массовых запросах возможен rate limit (HTTP 429) у `googlenewsdecoder`; при повторном сборе нужны паузы или меньше запросов за сессию.
4. **RSS РБК** недоступен — опора на скрапинг HTML и Google News `site:trends.rbc.ru`.
5. Корпус **не является** размеченным набором «мнений N экспертов» — это преимущественно СМИ и агрегаторы (REGIONS.ru, Интерфакс, региональные порталы).

## Как воспроизвести

1. Открыть `load_data.ipynb`, установить зависимости (ячейка 0).
2. В конфиге оставить только:
   ```python
   COLLECT = {"naukogrady": True, "rosatom": False, "international": False, "general_science": False}
   ```
3. Выполнить ячейки: импорты → конфиг/хелперы → сбор → сводка → граф.

Полный прогон сбора занимает **десятки минут** (задержка 0.3 с между запросами, сотни URL, NER/YAKE на каждый документ).
