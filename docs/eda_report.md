Сравнение графов

### Конструктор vs legacy

Конструктор **чуть разреженнее** (216 vs 317 рёбер): дополнительные фильтры (риторика, эмоции, paraphrase-кластер) и другой набор узлов. PMI чуть выше — остаются более «связные» пары.

### Baseline GraphRAG

Больше рёбер при меньшем числе узлов (только NER + co-occurrence на уровне документа/предложения), **меньше компонент** (30) — более «склеенный» entity-граф, меньше риторического слоя.

## 4. Центральные узлы и «жареные» связи

В конструкторе доминируют **топонимы и институты**: российской федерации, Саров, Дубна, Королёв, академии наук, научно-технологическое развитие — соответствует теме.

Рёбра с высоким **surprisal** (перплексия/surprisal-слой) — кандидаты на «неожиданные» связи для фазы 2; список: `output/eda/top_surprisal_edges.csv`.

Пример: пары с контрастом (`rhetorical`, relation=contrast) связывают, например, **Саров — Снежинск** (история атомных городов).

## 5. Ограничения и артефакты

1. **Шум в YAKE:** «автор текста», «главная общество» — не доменные концепты; в конструкторе частично сглаживается кластеризацией парафраз.
2. **Омонимия топонимов:** Королёв (город vs фамилия) — нужен контекстный фильтр.
3. **Много мелких компонент** (~63) — типично для sparse co-occurrence на 211 узлах; для reasoning полезны сообщества и топ-подграф.
4. **Мало экспертных голосов** — dropout по `actor` слабый (27 док.).

## 6. Как используются результаты EDA


| Результат EDA                 | Использование                                                |
| ----------------------------- | ------------------------------------------------------------ |
| `corpus_stats.json`           | Выбор порогов, оценка покрытия домена                        |
| `graph_metrics.json`          | Сравнение конструктора и baseline; контроль плотности ~2n–3n |
| `sanity_checks.json`          | Регрессия при пересборке корпуса / смене методов             |
| Фигуры `output/eda/figures/`  | Отчёты, слайды, промежуточный показ эксперту                 |
| `top_surprisal_edges.csv`     | Приоритет для интерпретатора «жареных» связей (фаза 2)       |
| Проверка voice_type / городов | Планирование сбора интервью и баланса источников             |


**Ноутбук** `eda_naukogrady.ipynb` — интерактивное воспроизведение; **скрипт** `scripts/run_eda.py` — однократная генерация всех артефактов для CI/отчёта.
