## Ресерч по проекту: методы, датасеты и аналоги

### 1. Какие методы будем использовать

Ключевые подходы и источники:

1. **Connected Concept Analysis (Lindgren, 2016)** — возьмём за основу, чтобы построить дискурс‑граф как сеть концептов (а не просто фактологический граф знаний). С ним получится:
* отбирать концепты;
* выстраивать связи — смотреть, в каких контекстах они появляются вместе;
* разбираться в устойчивых структурах.

2. **Semantic Network Analysis (Drieger, 2013)** — базовый способ построить семантическую сеть из неструктурированного текста. Здесь текст автоматически разбирается с помощью модели k‑next‑neighbourhood, а потом анализируется сетевыми методами.

3. **Discourse Network Analyzer (DNA, Leifeld)** — пригодится, чтобы проработать слой «актор – концепт – позиция». С DNA можно:
* аннотировать высказывания отдельных людей или организаций;
* выгружать сетевые матрицы — где акторы связаны общими концептами.

4. **TextRank (Mihalcea, Tarau, 2004)** — простой графовый метод для отбора ключевых слов и фраз. У него два плюса: не нужно обучать модель, а результаты легко интерпретировать.

5. **YAKE (Campos et al., 2020)** — подойдёт, чтобы извлекать ключевые слова из небольших наборов текстов или отдельных документов. Используем на первом этапе — когда надо отобрать вершины графа без обучения и без внешнего корпуса.

6. **KeyBERT** — поможет отобрать ключевые фразы на основе эмбеддингов. Метод ищет фразы, которые максимально близки к документу по BERT‑эмбеддингам. Чтобы снизить дублирование и повысить разнообразие кандидатов, можно добавить MMR (Maximal Marginal Relevance).

7. **RST Discourse Parser для русского языка (Ru‑RST)** — используем, чтобы выявлять риторические связи: контраст, уточнение, причину и т. д. Для русского есть готовые решения — например, IsaNLP RST Parser.

8. **GraphRAG (Edge et al., 2024)** — основной метод на базе GraphRAG. Он строит граф знаний по сущностям, потом формирует сводки по сообществам и использует их, чтобы отвечать на вопросы глобального осмысления.

9. **LightRAG (Guo et al., 2024)** — быстрый графовый RAG‑метод. Совмещает графовые структуры и векторные представления, использует двухуровневый поиск.

### 2. Чем подходы отличаются друг от друга

Коротко сравним основные варианты — посмотрим, что они умеют, какие у них плюсы и минусы:

| Подход | Что делает | Плюсы | Минусы |
|--------|----------|-------|-------|
| KG / GraphRAG | Строит граф сущностей и отношений (entity + relation graph) | Даёт фактологический QA, результаты легко интерпретировать | Плохо улавливает оценки и эмоции |
| Semantic Network Analysis | Строит граф совместного появления (co‑occurrence graph) | Просто и недорого | Много шумных связей |
| Discourse Network Analysis (DNA) | Анализирует связи «акторы + концепты + позиции» | Показывает конфликты и позиции | Требует дорогой разметки |
| RST / rhetorical parsing | Выявляет риторические отношения (contrast, cause и т. д.) | Улавливает аргументацию | Парсинг нестабильный |
| Long‑context LLM | Использует весь корпус в контексте | Не требует ручной настройки (zero engineering) | Может терять редкие, но важные связи |

**Где почитать подробнее:**
* про RST: https://www.sciencedirect.com/science/article/pii/S0957417420302451;
* про Discourse Network Analysis: https://eprints.gla.ac.uk/121525/;
* как превратить дискурс в сетевые связи: https://www.mdpi.com/2076-3417/13/12/6902;
* про семантический сетевой анализ: https://link.springer.com/article/10.1186/s40649-015-0021-8.

### 3. Методы для первой версии конструктора

**Как будем отбирать вершины:**
* NER (распознавание именованных сущностей);
* TF‑IDF;
* YAKE;
* TextRank;
* KeyBERT + MMR;
* кластеризация парафраз через эмбеддинги — потом присвоим каноническую метку с помощью LLM.

**Как построим связи:**
* по совместному появлению (co‑occurrence) — смотрим на предложения, абзацы или окна;
* взвешенное совместное появление — через PMI (Pointwise Mutual Information) или log‑likelihood;
* слой «актор – концепт – позиция»;
* риторические связи — ищем маркеры или используем RST;
* эмоциональный фильтр.

**Perplexity/surprisal** будем использовать не как основной генератор рёбер, а как инструмент для интерпретации неожиданных связей — уже после основной обработки.

### 4. Какие датасеты возьмём

**Основной датасет для первой итерации — «Наукограды России»:**
* **Почему он:** домен достаточно узкий, но в нём много разных сущностей — города, университеты, НИИ, корпорации, муниципальные и федеральные институты. Есть не только факты, но и оценки: автономия, федеральная поддержка, связь науки и бизнеса, городская среда для исследователей, научно‑производственный комплекс.
* **Контекст:** РБК описывает наукограды через численность исследователей и объём научной/наукоёмкой продукции. Недавно, в феврале 2026 года, Долгопрудному присвоили статус наукограда — это свежий инфоповод.
* Источник: [РБК Тренды][10].

**Финальный датасет для экспертной оценки — «Международное научно‑техническое сотрудничество»:**
* **Сузим тему:** «Международное научно‑техническое сотрудничество России: БРИКС, Азия, Африка, технологический суверенитет, постсанкционная кооперация».
* **Почему он:** у нас есть доступ к 20 экспертам, а основная метрика проекта зависит от того, насколько наш подход лучше GraphRAG/Gemini baseline. Тема актуальна — её обсуждают в контексте БРИКС, университетов, индустрии и технологического суверенитета.
* Источник: [issek.hse.ru][11].

**Контрольный датасет — «Росатом»:**
* **Статус:** не основной, нужен для проверки.
* **Почему он:** корпус уже известен, поэтому есть риск неявной оптимизации под заранее понятные инсайты. Будем использовать как sanity check — проверим, как интерпретатор находит «жареные» связи, и оценим, насколько воспроизводимы результаты старого кейса.
* **Контекст:** публичные интервью Росатома подходят для контроля — в них есть и факты, и оценки продуктов, технологий и потребителей.
* Источник: [strana‑rosatom.ru][12].

### 5. Итоговое решение

Для разработки конструктора и промежуточного показа возьмём **датасет «Наукограды России»** — он лучше всего подходит на старте.
